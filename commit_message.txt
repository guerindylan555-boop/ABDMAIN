feat: Performance and image optimization, CSS cleanup, and static page forcing

This commit focuses on improving performance, optimizing image loading, cleaning up CSS, and ensuring marketing pages are statically generated for faster loading.

Key changes include:

- **Performance — Images (weight & lazy loading)**:
  - Replaced `<img>` with `next/image` for avatars in `components/mvpblocks/testimonials-marquee.tsx` and for the logo in `app/components/Footer.tsx`.
  - Removed dependency on `randomuser.me`; avatars are now local (`/public/img/testimonials/avatar-default.svg`).
  - Modernized favicon by adding `/public/favicon.svg` and declaring it in `metadata` icons.

- **LCP — Gradient & Preload**:
  - Restored gradient rendering to CSS (`background-image`) in `app/layout.tsx` for visual fidelity.
  - Added explicit preload link for the gradient image in `app/head.tsx` (`<link rel="preload" as="image" href="/img/background/gradient-optimized.png">`).
  - Added a hidden `Next/Image` component in `app/layout.tsx` to force Next.js to optimize and preload the gradient image.

- **Image Configuration**:
  - Kept modern image formats (AVIF/WebP) in `next.config.mjs`.
  - Removed `remotePatterns` from `next.config.mjs` as `randomuser.me` dependency is gone.

- **Performance — TTFB: Forced Static Marketing Pages**:
  - Added `export const dynamic = 'force-static'` to all marketing pages and the homepage (`/solutions`, `/audit-custom`, `/offres`, `/offres-ia`, `/site-web-sur-mesure`, `/seo-local-google-business`, `/seo-google-business`, `/reservation-appel`, `/publicite-payante`, `/notre-methode`, `/hubs` and sub-pages, `/diagnostic-90-min`, `/a-propos`, `/automatisations-appels-sms`, `/blog`, `/blog/[slug]`).
  - This ensures faster TTFB due to pre-generated HTML and edge caching.

- **Performance — Blocking CSS: Decoupling and Lightening**:
  - Removed global `tw-animate-css` import from `app/globals.css`, replacing it with necessary local animations.
  - Removed global `react-day-picker/style.css` import from `app/components/ui/calendar.tsx` to avoid loading it unnecessarily.
  - `Scheduler` component is now client-only (`next/dynamic` with `ssr: false`) on `/audit-custom` to reduce initial bundle size.

- **Fix — Testimonials (avatars)**:
  - Resolved display regression where SVG avatars were not showing correctly in the `Marquee` carousel.
  - Reverted to native `<img>` for avatars in `components/mvpblocks/testimonials-marquee.tsx` with `loading="lazy"`, `decoding="async"`, `fetchPriority="low"`, and a fallback to `/img/testimonials/avatar-default.svg`.
